## その勉強法、科学的に間違っていませんか？

「テキストにハイライトを引く」「語呂合わせで覚える」「試験前日に詰め込む」。
これらは多くの人が実践していますが、認知科学の研究では **効果が低い** とされています。

Syllabus Hackでは、根性論を排除し、エビデンスに基づいた最強の学習法だけを採用します。

## 1. アクティブ・リコール (Active Recall)

「インプット」ではなく「思い出す」プロセスにこそ、学習効果の本質があります。

- **Why** : 脳は情報を「取り出す」時に回路を強化する。
- **How** : テキストを閉じ、何も見ずに「さっき読んだ内容」を書き出す・話す。
- **Tool** : 自作クイズアプリ、Notionのトグル機能。

## 2. 分散学習 (Spaced Repetition)

「忘却曲線」に逆らわず、忘れた頃に復習する。

- **Why** : 短期間の詰め込みは長期記憶に残らない。
- **How** : 復習間隔を 1日後 → 3日後 → 1週間後 → 1ヶ月後 と広げていく。
- **Tool** : Anki、自作ドリルの「忘却防止アルゴリズム」。

## 3. インターリービング学習 (Interleaved Practice)

1つの科目を集中してやるより、複数の科目を混ぜて学習する。

- **Why** : 脳に常に「切り替え」の負荷を与えることで、応用力が高まる。
- **How** : 「セキュリティ」だけをやり続けるのではなく、「セキュリティ」→「マネジメント」→「アルゴリズム」とランダムに取り組む。

## 4. 精緻化リハーサル (Elaborative Rehearsal)

丸暗記ではなく、既存の知識と結びつける。

- **Why** : 意味のない文字列は覚えられない。
- **How** : 「なぜそうなるのか？」を問いかけ、自分の言葉で説明する（プロンプトエンジニアリング的思考）。

## AIをコーチにする：スケジューリング・プロンプト

これらの手法は、単なる「コツ」ではなく **認知科学で実証された「理論」** です。
どれか一つでもピンときたものがあれば、AIに「学習スケジューラー」になってもらいましょう。

### A. 分散学習スケジュール作成

> 「これから2週間でITパスポートの『テクノロジ系』を攻略したいです。
> 分散学習（Spaced Repetition）の理論に基づき、初日の学習内容をいつ、どの間隔で復習すべきか、具体的なカレンダー案を作成して。」

### B. インターリービング混合メニュー作成

> 「今日の3時間で『セキュリティ』『ネットワーク』『法務』を勉強します。
> 脳に負荷をかけるインターリービング学習の観点から、20分ごとの学習順序をランダムに組み替えたタイムテーブルを作って。」

### C. アクティブ・リコール用の問題作成

> 「このテキストの内容を読んだ後、自分にセルフテストを行いたいです。
> 私が内容を『思い出す（Recall）』ために最適な、ひっかけ要素を含む確認問題を5門作成して。」

## まとめ：脳のOSをアップデートせよ

努力の量ではなく、努力の「質」を変えること。
これらの科学的手法を、生成AIという「伴走者」に管理させ、アプリなどのツールで習慣化するのが Syllabus Hack の真髄です。
根性論は捨てて、脳の仕様に合わせた最短ルートを突き進みましょう。
